三、现代操作系统四种基本观点

User/Computer Interface:是计算机用户使用计算机系统的接口，为计算机用户提供工作环境
Virtual　Machine:是建立在计算机硬件平台上的虚拟机器．为应用软件提供了许多比计算机硬件功能更强或计算机硬件所没有的功能
Resource Manager:是计算机系统中各类资源的管理者，他负责分配＼回收以及控制系统中的各种软硬件资源
Job Organizer:是计算机系统工作流程的组织者，他负责协调在系统运行中的各个应用软件的运行次序

四、现代操作系统需求

　功能性：软件系统能提供的服务：计算机用户需要的用户命令：表现形式：字符　菜单　图像

                                        　　　使用方式：脱机使用　联机使用
                    　　应用软件需要的system call，API:POSIX.1 WIN32

　非功能性需求：软件系统在提供这些服务时，需要满足的限制条件：performance or effciency
　　　　　　　　软件系统具有适应某些变化的能力

五、Thread线程

程序的一次相对独立的运行过程；是现代OS系统调度的最小单位．

　Process进程,系统分配资源的基本对象；在现代OS中，进程仅仅是系统中拥有资源的最小实体；在传统OS中，进程同时也是系统调度的最小　单位．
　Virtual　Memory：进程的逻辑地址空间．是现代OS对计算机中多级物理存储体系进行高度抽象的结果　
　File:命名的了字节流；是现代ＯＳ对计算机系统中种类繁多的外部设备进行高度抽象的结果．　

六、分时操作系统

七、现代OS的两个基本特征：

任务共行：宏观：指系统中有多个任务同时运行
　　　　　微观：指单处理机系统中的任务并发taskconcurrency即多个任务在单个处理机上交替运行或处理机系统中的任务并行，即多在多 个处理机上同时运行
资源共享：宏观：多个任务可以同时使用系统中的软硬件资源
　　　　　微观：多个任务可以交替互斥的使用系统中的某个资源

八、任务管理模型

　task：计算机系统在某个资源集合上所做的一次相对独立的计算过程
　mode:程序在运行过程中使用的有硬件体系结构提供的CPU特权模式

九、内存表：

　　记录内存的分配情况
　　记录外存分配情况
　　访问共享内存区的保护属性
　　管理虚拟存储需要的信息
　
　　I/O表：
　　io设备是可用的或已分配的
　　io操作的状态
　　io传输源和目的地在主存的位置

　　文件表：
　　有哪些文件存在
　　文件在外存的位置
　　文件的状态
　　文件的属性
　　
　　进程表：
　　进程的位置
　　管理进程的属性（id,name,state,location)
　　
　　进程位置：
　　进程包含一组程序：局部变量和全局变量的数据位置
　　　　　　　　　　　定义的常量
　　　　　　　　　　　堆栈：存放系统及过程调用地址参数
　　进程PCB：属性集合
　　进程映像：collections of program ,data,stack and attributes

十、PCB

(process control block):os控制和管理进程时所用的基本数据结构

　　作用：pcb是相关进程存在于系统中的唯一标志;系统根据pcb感知相关进程的存在（pcb一定在内存里）
　　进程标识：进程id
　　　　　　　父进程id　
　　处理器状态信息：用户可见寄存器
　　　　　　　　　　控制和状态寄存器
　　　　　　　　　　堆栈指针　
　　　　　　　　　　数据结构
　　　　　　　　　　进程特权

十一、操作系统内核功能：

　　资源管理功能：进程管理：进程创建和终止，调度，状态转换，同步和通信，管理pcb
内存管理：为进程分配地址空间，兑换，段/页管理
I/O管理：缓存管理，为进程分配io通道和设备
支撑功能：中断处理(interrupt handle)
　　　　　时钟管理(time)
　　　　　原语(primitive)：atomic operation,不可被中断，类比与原子不可分割
进程切换(process switch)
原因：clock interrupt
　io interrupt
memory fault存储访问失效
trap(when error occurred or may cause process to be moved to exit state)
supervisor call(such as file open)
过程：save context of processor including program counter and other registers
update the PCB of the process that is currently running
move PCB to appropriate queue-ready,blocked
select another process for execution
update the PCB of the process selected
update memory-management data structures
restore context of the selected process
创建与终止(create and terminate)
阻塞与唤醒(block and wakeup)
挂起与激活(suspend and activate)
　　　　　统计(account)
　　　　　监测(monitor)

十二、process switching vs mode switching

process switching作用于进程之间的一种操作．当分配程序收回当前进程的CPU并准备把它分派给某个就绪进程时，该操作将被引用
mode switching进程内部所引用的一种操作．当进程映像所包含的程序引用核心子系统所提供的系统调用时，该操作将被引用．

十三、创建进程过程

a.为进程分配一个唯一表示id：主进程表中增加一个新的表项
b.为进程分配空间：用户地址空间、用户栈空间、PCB空间。若分配已有空间，则应建立相应的链接
c.初始化pcb，处理机状态信息、进程状态
d.建立链接：若调度队列是链表，则将新进程插入到就绪或挂起链表
终止进程:原因:batch job issues halt instruction
user logs off
quit an application
error and fault conditions)

          normal completion正常结束
          time limit exceeded超市执行，执行时间超过预计时间
          memory unavaiable内存不足，无法为进程分配所需的内存空间　
          bounds violation越界访问
          protection error企图使用未允许用的数据或操作方式错
          arithmetic error计算错，如除０，或企图存储引荐允许的最大数
          time overrun超时等待某事件发生
          I/O failure如找不到文件或多次重试仍无法读取文件或无效操作
          invalid instruction企图执行不存在的指令
          privileged instruction企图执行特权指令
          data misuse数据类型不符，或未初始化
          operating system intervention操作ｉ员或OS干预，如发生死锁
          parent terminates so child processes terminate父进程终止所以子进程终止
          parent request父进程要求
     过程：a.根据id信号到pcb,读出该进程状态 
          b.若该进程为执行态，则终止其执行，调度新进程执行
          c.若该进程有子孙进程，则终止其所有子孙进程
          d.将该进程的全部资源或归还给其父进程，或归还给系统
          e.将被终止进程的pcb从所在队列中移出，等待其他程序来搜集信息
