1、Bean创建方式

1、使用默认构造函数

  • 在spring的配置文件中适用bean标签，配以id和class，且没有其他属性接标签时
  • 没有默认构造函数则创建失败

2、使用普通工厂中的方法创建对象

  • 使用某个类中的方法创建对象并存入spring容器

3、使用工厂中的静态方法创建对象

  • 使用某个类中的静态方法创建对象，并存入spring容器

2、Bean作用范围及生命周期

2、Dependency Injection

1、构造函数

  • 使用标签：constructor-arg
  • 标签中的属性:
    • type:制定要注入的数据点数据类型，该数据类型也是构造函数中某个或某些参数的类型
    • index:用于制定要注入的数据的数据 给构造函数中制定索引位置的参数赋值，索引从0开
    • name:用于指定给构造函数指定参数名称的参数赋值（常用的）
      ========以上三个用于指定给构造函数中哪个参数赋值=========
    • value:用于提供基本类型和String类型的数据
    • ref:用于制定其他的bean类型数据，即spring IOC容器中出现的bean对象
  • 优点:在获取bean对象是，注入数据是基本的操作，否则无法创建成功
  • 劣势:改变了bean对象的实例化方式，使我们在创建对象是，即使用不到一些数据也要对其赋值

2、Set方法

  • 使用标签：property

  • 标签属性：

    • name:用于指定注入时所调用的set方法名称

    • value:用于提供基本类型和String类型的数据

    • ref:用于制定其他的bean类型数据，即spring IOC容器中出现的bean对象

  • 优点：创建对象是没有明确的限制，可以直接使用默认构造函数

  • 缺点：如果某成员必须有值，获取对象时set方法可能没有执行

  • 复杂类型的注入/集合类型的注入
    • 用于给list结构集合注入的标签有
      • list
      • array
      • set
    • 用于Map结构集合注入的标签：
      • map
      • props

3、注解注入

  • 使用注解需要指定要扫描的包
  • Bean标签由@Bean注解代替

1、Bean标签的代替

  • @Component

         作用:
                用于把当期对象存入spring容器中
        属性：
                value:用于指定bean的id。当不写时，默认值为当前类名首字母小写

  • @Controller:一般用在表现层

  • @Service:一般用在业务层

  • @Repository:一般用在持久层

  • 以上三个注解的作用和属性与Component一模一样

  • 这三个是spring框架为我们明确提供的三层使用的注解，使我们的三层对象更清晰

2、数据注入

这三个用于注入其他Bean类型的数据

  • @Autowried

    • 自动按照类型注入

    • 只要容器中有唯一一个bean对象类型和要注入的变量类型匹配就可以注入成功

    • 如果ioc容器中没有任何bean的类型和要注入的类型匹配，则报错

    • 如果有多个匹配时,根据类型全定出匹配的对象，再根据变量名称作为id继续查找,无匹配时报错

    • 出现位置:变量或方法上

    • 细节:在使用注解时，set方法不是必须的

  • @Qualifier

    • 在按照类中注入的基础上(Autowired)再按照名称注入

  • 给成员注入时不能单独使用，但是在给方法注入参数时可

    • value：用于指定注入bean的id

    • @Resource

      • 直接按照bean的id注入
      • 可独立使用 =AutoWried+Qualifier
      • name:用于制定bean的id

3、集合类型注入只能通过XML实现

注入基本类型和String

  • @Value
    • 用于注入基本类型和String类型的值
    • value：用于指定数据的值，可以使用spring中的SpEL(spring的el表达式）
    • SpEL写法：${表达式}

作用范围和生命周期

  • @Scope

    • 用于制定bean的作用范围
    • value：singleton 单例
    • prototype 多例

  • @PreDestroy

    • 用于指定销毁方法

  • @PostConstruct

    • 作用:用于指定初始化方法

3、全注解配置

使用Java类配置

  • @Configuration

    • 指定当前类为配置类
    • 当配置类作为AnnotationConfigApplicationContext对象创建参数是，该注解可以不写

  • @ComponentScan

    • 通过注解指定spring在创建容器是要扫描的包
    • value，与basepackages的作用相同

  • @Bean

    • 吧当前方法的返回值作为bean对象存入spring的ioc容器
    • name：用于指定bean的id，不写默认值是当前方法的名称
    • 当我们使用注解配置时，如果方法有参数，spring在框架取容器中查找有没有可用的对象
    • 查找方法和Autowired注解的作用相同

  • @Import

    • 导入其他的配置类
    • value：指定其他配置类的字节码
    • 当使用Import的注解之后，有Import注解的类成为父配置类，导入的类都是子配置类

  • @PropertySource

    • 用于指定propertitiess文件的位置
    • value：指定文件的名称和路径
    • classpath：表示类路径下

4、动态代理

字节码随用随创建，随用随加载
不用修改源码即可进行增强

1、Proxy代理

基于接口的动态代理，由JDK支持

如何创建：使用Proxy类中的newProxyInstance方法

要求：被代理类最少使用一个接口，如果没有使用则不能用

2、Cglib代理

基于子类的动态代理，第三方cglib类Enhancer

要求：不能是最终类

如何创建：使用Enhancer类中的create方法
