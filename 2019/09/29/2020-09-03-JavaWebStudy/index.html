四种范围对象：
pageContext:jsp页面容器        当前页面有效，跳转后无效　
request:请求对象　            同一次请求有效(请求转发有效，重定向后无效)
session:会话对象　            同一次会话有效(无论怎么跳转都有效;关闭/切换浏览器后无效，从登录到退出都有效)
application:全局对象            全局有效(切换浏览器仍然有效)关闭服务器、其他项目无效
(想要多个项目共享、重启仍然有效：JNDI)
以上四个对象共有方法：
    Object getAttribute(String name):根据属性名或属性值
    void setAttribute(String name,Objetc obj):设置属性值，可新增或修改
    void remove(String name):根据属性名删除对象
以上范围对象尽量使用最小的范围，减小开销




response:响应对象
    void addCookie(Cookie cookie)：服务端向客户端增加cookie对象
    void sendRedirect(String location)　throws IOException:页面跳转的一种方式:重定向
    void setConteType(String type):设置服务端响应的编码(设置服务端的contentTypele类型)

request:请求对象，存储客户端向服务端请求的服务信息，只在同一次请求有效，当地址栏回车时，发出第二次请求，如果点击F5刷新，则浏览器会重复刚才的行为

session：会话
    类型：
    a.浏览网站：开始到关闭
    b.购物:浏览　付款　退出
    c.电子邮件：浏览　写邮件　退出

    机制：　　
    客户端第一次请求服务器时，(匹配失败，分配一个session)服务端会产生一个session对象（用于保存客户的信息）
    每个session对象都有一个sessionId(用于区分其他session)
    服务端会产生一个cookie，该cookie的name=JSESSION,value=服务器sessionId的值
    因此客户端cookie与服务端的session一一对应　
    客户端在第二/N次请求服务端时：服务器会用客户端cookie去匹配中的JSESSIONID副服务端的sessionId：
        如果匹配成功(cookie jsession 和session sessionId).说明次用户不是第一次反问该服务端，无需再登录
        匹配失败
        例子：
        客户端：        顾客
        服务端：　    商场
        顾客第一次存包，商场判断此人之前是否存在过包（通过你手中是否有钥匙判断）如果是新顾客（没钥匙），分配一个钥匙给顾客，钥匙与柜子一一对应
        顾客第二次存包，商场判断此人之前存过包（通过手中是否有钥匙）如果是老顾客，则不需要分配，该顾客手中的钥匙会与柜子一一对应）　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　
    性质：
    a.session存储在服务端
    b.session是在同一个用户请求时共享
    c.实现机制：第一次客户请求时，产生一个sessionId并复制给cookie的jsessionid然后发给客户端。最终连通

15.JDBC:Java Database Connectivity,可为多种关系型数据库DBMS提供统一的访问方式，用java操作
JDBC DriverManager:管理不同的数据库驱动　
JDBC API:

       作用：
        与数据库建立连接
        发送SQL语句
        返回处理结果

       通过一下类/接口实现：
        DriverManager:管理jdbc驱动
        Connection：连接(通过drivermanager产生)

        Statement：增删改查（通过connection产生：connection.createStatement()方法
        Statement操作数据库：
        增删改：executeUpdate()
        查询:executequery()

        PrepareStatement操作数据库：增删改查（通过connection产生：connection.prepareStatement()
        public interface PreparedStatement extends Statement
        因此
        增删改：executeUpdate()
        查询:executequery()
        赋值操作：setXxx()

        CallableStatement：调用数据库的存储过程/存储函数（通过connection产生:
            prepareCall(参数：存储过程或存储函数名)
            参数格式：存储过程：无返回值，用out参数代替
                            {call 存储过程名（参数列表）}
                    存储函数：有return返回值
                            { ? = call 存储函数名（参数列表）}
        create or replace procedure addTwoNum(num in int,num2 in int,result out int)
        as
        bengin
        result:=num1+num2;
        end;
        /

        通过sqlplus访问数据库，只需要开启：OracleServiceSID
        通过其他则需要访问数据(such as sqldeveloper,JDBC),需要开启：OracleServiceSID,XxListener

        PreparedStatement与Statement区别：
        statement： sql
                    executeUpdate()
        PreparedStatement:sql(可能存在占位符)
                      在创建ps对象时，将sql预编译：PreparedStatement(sql)
                      setXxx()替换占位符
                      executeUpdate()

        开发推荐使用preparedStatement：
        1.编码更简便(避免字符串拼接)
        String name = "zs";
        int gae = 23;

        stmt:
        String sql = "insert into student (stuno,stuname) values ('"+name+"',"+age+")";
        stmt.executeUpdate()

        pstmt:
        String sql = "insert into student (stuno,stuname) values (?,?)";
        pstmt = connection.preparedStatement(sql); //预编译sql
        pstmt.setString(1,name);
        pstmt.setInt(2,age);

        callableStatement:
        cstmt = connection.preparecall(); //产生调用存储过程的对象
        cstmt.setXxx(1,30) //通过set处理输入参数值
        cstmt.set(3,Types.INTERGER) //通过set设置输出参数类型
        cstmt.execute(); //执行
        rs = cstmt.get(3); //获取输出值
        sout(rs); //打印输出值

        2.提高性能（有预编译操作）
        需要重复增加100次
        stmt:
        String sql = "insert into student (stuno,stuname) values ('"+name+"',"+age+")";
        for(100)
            stmt.executeUpdate(sql) //sql编译100次

        pstmt:
        String sql = "insert into student (stuno,stuname) values (?,?)";
        pstmt = connection.preparedStatement(sql); //预编译sql
        pstmt.setString(1,name);
        pstmt.setInt(2,age);
        for(100)
            pstmt.executeUpdate() //sql只用编译一次

        3. 安全(有效防止sql注入:将客户 输入内容与sql语句混为一体)
        stmt存在被sql注入的风险：例如输入用户名：abc(任意值) ’ or 1=1 -- ,密码：def(任意值)

        Result:返回的结果集（通过statement产生）
    next():下移光标，判断下移之后的元素是否有数据，有返回true，没有返回false
       previous():下移光标，判断下移之后的元素是否有数据，有返回true，没有返回false
       getXxx(字段名or位置下标):获取指向行的数据


       访问过程：
           try{
        a.导入驱动，加载具体驱动类Class.forName()
        b.与数据库建立链接connection = DriverManager.getConnection()
        c.通过connection获取操作数据库的对象（Statement/preparedStatement/callableStatement）
        stmt = connection.createStatement()
        d.(查询需要)处理结果集 rs = pstmt.executeQuery()
        where(rs.next()) {rs.getXxx()}
        }catch(ClassNotFounException e){...} //class.forName()抛出
        catch(SqlException){...}  //sql语句抛出
        catch(Exception e) {...}
        finally{
            //关闭顺序与打开顺序相反
            if(rs != null)rs.close();
            if(stmt != null) stmt.close();
            if(connection != null)connection.close();
        }

C:controller控制器:接受请求，将请求跳转到模型进行处理，处理结果返回给控制器。可以用jsp实现，但建议用Servlet实现控制器

20.三层架构(与MVC的目标一致:为了解耦合，提高代码复用.但二者对项目理解不同)
表示层:(USL,UserShowLayer;视图层)
-前台:对应MVC中的View，用于交互和界面显示
jsp js html css jQuery等前端技术,Webcontent中
-后台:对应MVC中的Controller,用于页面控制跳转，调用业务逻辑层
Servlet (SpringMVC Struts2 …),位于Xxx.servlet包中

业务逻辑层:(BLL,BusinessLogiclayer;Service层
-组装数据访问层，逻辑性操作(增删改查，删：查+删) 位于Xxx.service包中or Xxx.manager

数据访问层:(DAL,DataAccesslayer;Dao层)
-直接访问数据库，原子性操作(增删改查) 位于Xxx.dao包中

三层之间的关系:上层将请求传递给下层,下层处理后返回给上层.上层依赖于下层
依赖:持有成员变量

三层优化:
加入接口,建议面向接口开发：先接口，再实现类:service,dao加入接口
接口与实现类的命名规范:
接口:interface 起名:IXxxService IXxxDao
接口所在包:xxx.service
实现类:implements 起名:XxxServiceImpl XxxDaoImpl
实现类所在包:xxx.service.impl
接口 x = new 实现类(); 例子:
IStudentDao studentDao = new StudentdaoImpl();

DBUtil数据库帮助类,简化Dao层的代码量
建议写在一个util包中

方法重构:将多个方法的共同代码提炼出来，单独写在一个方法中

select * from student where sid >=(n-1)10+1 and sid <=n*10; //存在漏洞，必须id值连续

select * from (select rownum, t.* from

select * from (
select rownum r,t. * from

rownum <= n*10
