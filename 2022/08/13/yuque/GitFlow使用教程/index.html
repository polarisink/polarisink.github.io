GitFlow使用教程

实习的时候使用的 git 管理方式

1、clone 项目到本地

不能使用 fork，必须 clone
fork 用于载服务器端建立副本，clone 创建项目的本地副本

2、初始化 gitflow

每次 clone 存储库都需要这么做

3、创建特性分支

所有的工作（新特性、不紧急的 bug 修复）都必须在功能开发分支进行
特性分知名需要得体，可以使用代号为功能开发分支命名
如果主存储库一级存在该功能开发分支，命令会失效

4、在 github 发布功能开发分支

5、不断更新

6、和别人协同开发

7、合并你的分支到 develop 分支

8、创建测试分支

9、创建热修复
